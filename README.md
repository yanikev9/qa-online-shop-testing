# QA Online Shop Testing Project

## Project Description
This project demonstrates manual and API testing of a demo online shop application.

The goal of this project is to practice QA testing techniques and create a testing portfolio including test documentation and bug reports.

---

## Scope of Testing

The following functionality was covered during testing:

- User registration
- User login
- Cart functionality
- Delivery price calculation
- Order checkout process
- API registration endpoint

---

## Types of Testing Performed

- Functional testing
- Negative testing
- Boundary value analysis
- Equivalence partitioning
- Regression testing
- API testing

---

## Tools Used

- Postman
- Chrome DevTools
- SQL
- GitHub for documentation

---

## Test Documentation

This project contains the following QA artifacts:

### Checklists
Checklist for registration and login functionality.

### Test Cases
Detailed authentication test cases with steps and expected results.

### Bug Reports
Documented bugs discovered during testing.

### API Testing
API test scenarios for the `/register` endpoint.

---

## Found Bugs

During testing the following issues were discovered:

1. Incorrect delivery cost displayed during checkout.
2. Account is not locked after multiple failed login attempts.
3. Registration API returns success response but user is not created in the database.
